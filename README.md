# 🎮 Game Database Management System (SQL)

## 🚀 Overview
This project is a relational database system designed for managing players, characters, clans, and games.  
It includes advanced SQL concepts such as stored procedures, triggers, and transaction-like rollback logic.

## 🧠 Features
- Relational database design with multiple tables  
- Stored procedures for querying and aggregation  
- Trigger-based logging system  
- Rollback mechanism using stored procedures  

## 🛠 Technologies
- SQL (MySQL)

## 📂 Project Structure
- `ister1.sql` → table creation + basic procedure  
- `ister2.sql` → credit calculation procedure  
- `ister3.sql` → trigger for level change logging  
- `ister4.sql` → rollback mechanism  

## 🔍 Key Functionalities

### 📌 High Level Character Query
Retrieves characters above a certain level using stored procedures.

### 📌 Clan Credit Calculation
Calculates total credit balance of a clan using OUT parameters.

### 📌 Level Change Logging
Tracks character level updates using triggers.

### 📌 Rollback System
Reverts character level changes using stored procedures and logs.
